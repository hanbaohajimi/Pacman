Architecture 03 · 算法骨架
FOUR
一个接口，四种展开
DFS 深度优先
栈 · 一路到底
使用栈，沿一条分支不断深入。空间占用小，能快速找到解，但无法保证最优路径。bigMaze 路径不稳定。
路径: 不保证最优
BFS 广度优先
队列 · 逐层扩散
使用队列，逐层展开，等代价时严格保证最短路径。mediumMaze 扩展 269 节点，路径 68 步最优。
路径: 等代价最优
UCS 均匀代价
优先队列 · g(n)
按累积代价 g(n) 排序。等代价图退化为 BFS，扩展节点完全相同，但需维护堆结构，略慢于 BFS。
路径: 任意代价最优
A★ 启发搜索
优先队列 · f=g+h
按 f(n)=g(n)+h(n) 排序。可采纳启发式保证最优解，mediumMaze 仅扩 221 节点，比 BFS 少 17.8%。
路径: 最优且最高效
// Graph-Search   push start → fringe  |  bestCost[start]=0  |  while fringe: pop → check goal → expand → push if better cost

Heuristics 05 · 启发式设计与证明
THREE
三个问题 · 三种证明
P1单点寻路
h(p) = |x−gx| + |y−gy|
曼哈顿距离，可采纳且一致，引导 A* 向目标方向收敛。
P2角落巡访
h = min d(pos,c) + MST(R)
到最近未访角落 + 剩余角落最小生成树权重，mediumCorners 扩展 774 节点。
P3全食物搜索
h = max mazeDistance(pos, f)
对所有剩余食物取最远迷宫距离（BFS 预计算），比曼哈顿更紧凑的下界。
mediumCorners · A* 角落巡访 · Score −26
"可采纳启发式保证 A* 展开必要节点，
比 BFS 最多减少 18% 扩展量。"
