# 吃豆人搜索实验 — 使用说明

本项目基于 UC Berkeley AI Pacman 框架，实现 BFS、DFS、UCS、A* 等搜索算法，并提供图形化启动器。

---

## 环境要求

| 项目 | 说明 |
|------|------|
| 操作系统 | Windows / macOS / Linux |
| Python | 3.8 ~ 3.12 |
| 依赖 | 仅 Python 标准库，无需 pip install |

---

## 快速开始

### 方式一：GUI 启动器（推荐）

```powershell
cd source
python pacman_launcher.py
```

启动后出现图形界面：
- **开始游戏** → 进入模式选择页，按分区浏览所有算法与地图
- **实验数据对比** → 查看 BFS / DFS / UCS / A* 在不同地图下的性能数据
- **游戏说明** → 操作方式、AI 可视化含义、计分规则

选择模式后点击 **"启动"**，Pacman 游戏窗口自动弹出。

### 方式二：命令行直接运行

```powershell
cd source

# 键盘游玩
python pacman.py -l tinyMaze

# A* 搜索
python pacman.py -l mediumMaze -p SearchAgent -a fn=astar,heuristic=manhattanHeuristic

# 四角问题
python pacman.py -l mediumCorners -p AStarCornersAgent

# 自动批改
python autograder.py --no-graphics
```

游戏窗口底部信息栏右侧显示实时计时器（`Time: MM:SS`），从游戏画面初始化开始计时，所有模式均适用。

## 游戏说明

### 操作方式

- **键盘模式**：方向键，或 **W/A/S/D** 移动，**Q** 停止
- 需使用图形界面（勿加 `--textGraphics`）
- **AI 演示**：在启动器中选择算法模式后点击「启动」

### AI 搜索可视化（红色方格）

适用于 **BFS / DFS / UCS / A*** 迷宫搜索（`SearchAgent`）。找到路径后，棋盘上会出现红色半透明方格，表示搜索过程中**扩展过的格子**：

- **颜色越深**：越早被扩展
- **颜色越浅**：越晚被扩展

多豆 Food 搜索（`AStarFoodSearchAgent`）默认不显示该 overlay。

### 得分与扣分

| 事件 | 分数 |
|------|------|
| 吃小豆 | +10 |
| 吃完所有小豆 | +500（胜利） |
| 每步 | -1（时间惩罚） |
| 吃能量豆 | 无直接分；幽灵惊吓 40 步 |
| 惊吓态撞幽灵 | +200 |
| 被正常幽灵撞到 | -500（失败） |

### 能量豆说明

地图上较大的白色圆点为**能量豆**（布局字符 `o`）。吃能量豆后幽灵进入惊吓状态（变白），可反吃幽灵（+200 分）。动态避障模式会在未惊吓幽灵较近时主动前往能量豆，吃到能量豆后会追击变白幽灵。

### 地图说明

- **testSearch**：Berkeley 批改用，仅 1 颗豆，非多豆演示
- **multiFoodTest**：约 10 颗豆的小型多豆地图，适合 Food 搜索演示（勿用上百颗豆的大图，否则会长时间无响应）

## 常见问题

**Q: 启动器点击"启动"后没反应？**
确认当前目录是 `source/`，Python 版本 ≥ 3.8。

**Q: 游戏窗口一闪就关？**
在终端中运行，不要直接双击 `.py` 文件。

**Q: Food / Corners 模式很慢？**
状态空间很大，A* 可能需要数秒到数十秒，属正常现象。

**Q: 手动经典大地图走一步卡一下（吃豆人/幽灵都卡）？**
与地图画风无关，常见原因：① 每按一次键，吃豆人动一步后每个幽灵各动一步，原先每步多次全画布 `refresh`；② 幽灵回合反复 `deepCopy` 整张地图；③ 动画帧间 `sleep` 在 Windows 上会嵌套 Tk `mainloop`。当前版本已合并每回合刷新、只读智能体跳过整盘复制、迷宫布局不再每步复制、动画改用轻量等待。画面仍为原版圆角墙与缩放。若仍觉慢，可在命令行加 `--frameTime 0` 去掉步间动画等待（仅影响移动节奏，不改地图样式）。
